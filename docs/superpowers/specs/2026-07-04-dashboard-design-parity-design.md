# 대시보드 디자인 패리티 — 설계 문서

작성일: 2026-07-04
상태: 승인됨 (구현 플랜 대기)

## 배경

Stitch 디자인(프로젝트 `3232228151373723761`)의 5개 화면(Summary, Feed, Trends,
Calendar, Admin)에 있는 위젯/기능 중 상당수가 현재 구현 사이트에 빠져 있다.
현재 사이트는 이미 **실제 DB(Drizzle/Postgres)** 로 동작하며 5개 라우트가 모두
존재하지만, 디자인의 개별 위젯이 누락된 상태다. 원본 디자인 파일은
`docs/`(핸드오프) 및 스크린샷으로 확보됨. Gap 감사 결과 빠진 위젯 대부분은
**기존 테이블로 지금 구현 가능**하다.

## 목표

디자인에 있으나 빠진 위젯/기능을, 데이터가 존재하는 범위에서 실제 동작하도록
구현한다. 한국어 UI + 기존 디자인 토큰/컴포넌트를 유지하되, 디자인의
레이아웃·위젯 구조를 충실히 반영한다.

## 디자인 충실도 방침

- 한국어 카피 유지, 기존 Tailwind 디자인 토큰/컴포넌트(`components/ui.tsx`,
  `Card`, 차트 컴포넌트) 재사용.
- 디자인의 **섹션 구성·위젯 종류·정보 밀도**를 따른다(레이아웃 파리티).
- 픽셀 단위 색/폰트 복제는 목표 아님.

## 범위 (구현)

### A. Summary
1. KPI 카드 **증감 배지** — 각 KPI에 전주/전일 대비 델타(방향+%). 신규 쿼리로 기간
   비교 계산.
2. **팔로워 성장 카드** — `accountMetricsDaily.followers` 를 플랫폼별로 집계한
   시계열 바/라인 차트 + 플랫폼별 현재 팔로워 + 집계 델타(%).
3. **최근 변경 타임라인 이벤트 타입화** — 최신 포스트 나열 대신, 파생 이벤트
   (신규 광고 감지 / 광고 비활성화 / 팔로워 급증)를 타입별 아이콘·색·썸네일로
   렌더. `ads.firstSeen/isActive`, `adPresenceDaily`, `accountMetricsDaily` 에서 파생.
4. 브랜드 현황 테이블은 기존 유지(감성/광고비 컬럼은 제외 — 아래 OUT).

### B. Feed
5. **필터 확장** — 기존 플랫폼/종류 외에 **브랜드 드롭다운**, **포맷 토글
   (image/video/carousel)**, **기간 선택(최근 7/30/90일)** 추가. `getFeed()` 및
   클라이언트 필터 확장.
6. **텍스트 인용형 카드 변형** — 미디어 없는 포스트는 캡션 인용 스타일 카드로.
   (투표 카드는 OUT.)
7. 캐러셀 카드에 매수 인덱스 배지(예: 1/5) — mediaAssets 개수 기반.

### C. Trends
8. **평균 인게이지먼트% 지표** — `accountMetricsDaily.engagementRate30d`(이미 존재,
   미사용) 를 헤더 카드에 노출.
9. **7D/30D/90D 범위 토글 + 라인차트 hover 툴팁** — 팔로워 시계열에 기간 토글과
   포인트 툴팁.
10. **주간 인게이지먼트 바차트** — `postMetricsDaily` 를 주(W1~W4) 단위 집계.
11. **성과 상위 포스트 테이블화** — 게시일 / 인게이지먼트 / 도달(도달은 없으므로
    조회수로 대체) / **상태(VIRAL·STABLE)** 컬럼.
12. **벤치마크 상태 카드** — 카테고리 내 백분위(정의는 아래).

### D. Calendar
13. **월간 히트맵 그리드** — 기존 날짜 리스트를 Mon–Sun 7열 월간 캘린더 +
    강도(Less→More) + 일자별 플랫폼 점으로 재구성. 월 선택기.
14. **콘텐츠 포맷 믹스 카드** — `posts.format` 그룹 집계(비율 바).
15. **주간 게시 빈도(posts/wk)** — 플랫폼별 주간 빈도 집계.
16. **하단 KPI 벤토** — 총 게시(MTD), 평균 일일 케이던스, **인게이지먼트 피크
    시간**(postedAt 시(hour) 추출). "활성 캠페인"은 OUT.
17. 최적화 팁 카드 — 계산된 값 기반 문구(예: 특정 요일 감소).

### E. Admin
18. **자동 새로고침(45s)** — 라우트 revalidate 폴링 + "N초 후 새로고침" 표시.
19. KPI 카드에 **전일 대비 델타 + 프로그레스 바**.
20. **로그 검색** — 브랜드/플랫폼/에러 텍스트로 클라이언트 필터.

## ⚠️ 계산 정의 (신규 지표)

- **벤치마크 백분위 (Trends #12):** 모니터링 대상 전 브랜드(현재 단일 뷰티
  카테고리)의 계정별 `engagementRate30d` 분포에서 대상 계정의 백분위를 계산.
  표시: 백분위 + 밴드(상위 10% → "High", 10–50% → "Mid", 그 외 "Low").
  데이터 부족 시(계정 <3) "N/A" 노출.
- **VIRAL / STABLE 상태 (Trends #11):** 각 상위 포스트의 인게이지먼트
  (likes+comments+views 합)를 해당 계정 포스트 인게이지먼트의 **중앙값과 비교**.
  중앙값의 **3배 이상 → VIRAL**, 그 외 → STABLE. 표본 부족 시 STABLE 기본.
  임계값은 상수로 두어 조정 가능.

## 범위 밖 (OUT — 데이터 없음)

- **감성분석(Sentiment)** — 분석 파이프라인/스키마 없음.
- **광고비 지수 / 도달 / 노출** — 경쟁사 광고는 비공개, 수집 불가.
- **활성 캠페인** — 캠페인 엔티티 없음.
- **투표(poll) 카드** — poll 데이터 없음.
- **콘텐츠 분류 태그** — `contentTags` 는 stub(파이프라인 없음).

이 항목들은 UI에서 **표시하지 않는다**(더미/예시 배지도 달지 않음). 해당 자리는
비우거나 남는 위젯으로 균형을 맞춘다.

## 아키텍처 / 영향 범위

- **백엔드(쿼리)**: `apps/web/app/lib/queries.server.ts` 에 신규 집계 함수 추가
  (KPI 델타, 플랫폼별 팔로워 시계열, 최근 변경 이벤트, 벤치마크 백분위, 주간
  인게이지먼트, 포맷 믹스, 피크시간, admin 델타 등). 스키마 변경 없음.
- **프론트**: 각 라우트 `.tsx` 와 공용 차트/카드 컴포넌트 확장. 필요한 신규
  프리젠테이션 컴포넌트(월간 히트맵 그리드, 범위 토글 차트, 이벤트 타임라인,
  포맷 믹스 바, KPI 델타 배지)는 `components/` 에 작게 분리.
- **데이터 소스**: 전부 실제 DB. 신규 스키마/마이그레이션 없음.

## 검증 기준 (DoD)

- [ ] 각 화면의 IN 항목 위젯이 실제 DB 값으로 렌더(빈 데이터 graceful).
- [ ] 벤치마크/VIRAL 정의가 상수화되어 테스트 가능.
- [ ] `npm run typecheck -w @celine/web` 및 `npm run build -w @celine/web` 통과.
- [ ] 모바일(390x844)에서 신규 위젯 가로 오버플로 없음(기존 반응형 원칙 유지,
      `lg:` 스코프 유지, `notranslate`/아이콘 규칙 준수).
- [ ] OUT 항목은 UI에 노출되지 않음.

## 열린 결정 (기본값)

1. 디자인 충실도 → **기능/레이아웃 파리티, 한국어+기존 토큰 유지** (기본값).
2. 데이터 없는 위젯 → **숨김(더미 없음)** (기본값).
3. 진행 순서 → **Summary → Feed → Trends → Calendar → Admin** (기본값).
